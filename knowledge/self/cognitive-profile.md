# 相談内容

「私の脳の癖を見つけたい、私を研究して」という依頼。一次データは
knowledge/ の相談記録16件（すべて 2026-07-06 の1日分）、
lets-deep-agents の失敗カタログ（28モード・7クラスタ）と log.tsv、自作 Stop hook 2本。

# 抽出した癖（6つ）

1. **抽象階段を一段上る** — 質問が常に道具の一段上を向く。「Claude Code の使い方」でなく
   「ハーネスとは何か」、「スクラムのやり方」でなく「AI時代に何が残るか」。
   最終的に [モデル非依存の原則集](/tech/ai-agent-mastery-essence.md) まで昇りきる。
2. **違和感を研究テーマに変換する** — 「ハーネス感がない」「何もやってない感覚」という
   言語化しにくい感覚を放置せず相談として定式化し、分類と機構で決着させる。
   感情の処理経路が体系化である。
3. **善意より機構を信用する** — 「モデルが最悪の気分の日でも守られるか」が口癖級の判定基準。
   自分自身にも適用し、記録の約束ではなく記録を強制する Stop hook を作った
   （[強制点の設計](/tech/skill-to-harness-enforcement.md)）。
4. **儀式アレルギー** — 形だけ回って機能しないものへの感度が非常に高い。
   儀式化した自己チェック（[要件skill評価](/tech/requirements-skill-review.md)）、
   「やってる感」ループ（[なんちゃってアジャイル分析](/thinking/fake-agile-systems-analysis.md)）、
   スクラムの儀式論。常に「この形は成果に因果的に配線されているか」を問う。
5. **失敗を一次データにする経験主義** — ルールは失敗を2回観測してから。失敗カタログでは
   検証の認識論的失敗（C4）が7モードで最大クラスタ。正しさより先に
   「不確実性が正直に報告されるか」（C7）を問題にする。60点で走らせて埋める方式を受容。
6. **消失への備え（外部脳の構築）** — OKF knowledge・tech-digest の蒸留・dotfiles・
   「Fable が使えるうちに申し送りを」という依頼。モデル・セッション・自分の記憶の
   揮発性を前提に、持ち越せる資産へ変換し続ける。

# 影（構造的な緊張）

- **メタ勾配**: workspace のほぼ全プロジェクトが「エージェントを使うためのエージェント研究」。
  メタ成果物は「これを私が作った」という物的な手応えを返しにくく、
  「作ってもらっただけ」の空虚感（[学び方の相談](/tech/learning-without-building.md)）は
  怠慢ではなく仕事の階層構造から来ている可能性が高い。検証法: エージェントと無関係な
  対象レベルの制作物を1つ作り、空虚感が変わるか観測する。
- **隠匿仮説の初期値**: スキル差を「情報が開示されていない」と解釈する初期傾向
  （[ハーネス基礎](/tech/ai-agent-harness-basics.md)の相談動機）。ただし証拠提示で即更新する。

# インタビュー編（同日追加・3ラウンド12問）

本人の指摘「行動分析は真面目な面に偏る」を受けて実施。行動ログ由来の仮説が2つ反証され、
プロファイルの中心が書き換わった。

## 反証されたもの

- ×「感情の処理経路が体系化」→ ストレス対処は寝る・食べる・体を動かす（身体系）。
  体系化は仕事領域限定の道具。
- ×「構造・仕組みへの美的嗜好」→ 惹かれる作品は人間ドラマ・王道熱血・雰囲気没入系で、
  「世界の仕組みが明かされる系」は非選択。美意識の核は感情と人間。

## 中心発見: 体系化は「AI以後の外骨格」

- 分類・体系化スタイルは**AIを使い始めてから**の獲得形質（子供の頃・社会人習得を非選択）。
- 機構で自分を縛る動機は「**自分の意志力を信用していない**」（工作の楽しさ等は非選択）。
- 自然体: 昼食は気分で直感 / 事務手続きは後回し / アイデアは散歩・風呂・深夜・手を動かす最中
  （拡散モード・身体駆動）/ 対人エネルギーは相手と場次第。
- したがってハーネス哲学「モデルが最悪の気分の日でも守られるか」は
  **「私が最悪の気分の日でも」の投影**。自分の認知的弱点（意志力・忘却・直感の暴走）の
  自己モデルを LLM に転写したことが、ハーネス設計の勘の良さの源泉である可能性が高い。

## 価値観の核は認識論ではなく倫理

- 理屈抜きにイラッとする対象: 人の時間を軽く扱う人・人の気持ちを考えない人。
  「根拠なき断定」「思考停止」は非選択。
- 失敗カタログ C7（報告の不誠実さ）の重視も、真理愛ではなく
  「不正確な報告は読み手への不敬」という思いやり駆動と解釈するのが整合的。
- 他者からの第一印象（自己申告）: 優しい・気を遣う人（+真面目・マイペース・凝り性）。

## 感情の消化器官は「書くこと」

作品で泣いた/熱くなったとき: 一人で味わい、**感想を書きたくなる**（人に語るのは非選択）。
knowledge/ への蒸留衝動は消失への備えだけでなく、感情・経験の native な消化経路。

# 投影法編（2026-07-07 追加）

刺激: 単語連想10語・文章完成6問・情景課題（手紙と鍵の部屋）・建物メタファー。
事後質問（インクワイアリー）3問を実施。※投影法は仮説生成器であり確定器ではない。

## 得られた素材

- 連想10語（機械・約束・夜・家族・失敗・水・仕事・扉・声・未来）**すべてに共通する1語**
  として「アンドロイド」。※本人は課題を「全項目に共通する答えを探す」と解釈した
- 文章完成6問**すべてに通る1フレーズ**として「この時代にいない」
- 情景課題 → 誤認逮捕された主人公が取調室におり、警察内部の誰かが**異変に気づいて**
  密かに逃がそうとしている物語。同一化先は登場人物ではなく**作者・観客**。
- 建物 → 超高層ビルの**30%の高さ**にいる。内装はナチュラルで落ち着いた空間。
  他の人は「考えていなかった」（拒絶ではなく画角の外）。
- 回答行動: 課題指示が曖昧なとき、確認せず「**全項目を貫く不変量を探す**」という
  （より難しい）解釈を自然に選んで実行した。個別列挙ではなく圧縮・共通構造の抽出が
  デフォルトの認知動作である証拠。
  ※初回解釈「列挙課題を退屈で飛ばした（マイペース）」は誤読につき**撤回**。

## 解釈仮説（他の方法で要検証）

1. **カメラ位置の人**: 自作の空想の中でさえ主人公でも救助者でもなく作者・観客を選んだ。
   仕事(オーケストレーター)・学習(判断者)・感情(一人で味わう)・物語(観客)のすべてで
   「見る位置」に立つ一貫性。強みであると同時に、「作ってもらっただけ」の空虚感の根と
   同じ構造 — カメラには「生きた・作った」の手応えが蓄積しない。
2. **恐れ = システムに誤って判定されること**（誤認逮捕）。「作ってもらっただけ」の
   自己疑念（本当の作者と認められない不安）との接続を疑う。
3. **英雄像 = 異変に気づいて静かに動く内部者**。観測と思いやりの複合 — 本人が
   失敗観測・正直な報告で訓練している役割そのもの。世界観は「制度は間違えるが、
   中の個人は善い」という希望型。
4. **「この時代にいない」が6つの文すべてに通る**: 本当の私は/他人は私を/見られていなければ/
   一番怖いのは/子供の頃/十年後 — どの文頭にも接続する自己規定として出てきた。
   帰属の軸が場所や集団でなく**時代**であり、「一番怖いのは、この時代にいない(こと)」と
   読むと疎外と喪失恐怖が両義的に共存する。モデル入替への備え・申し送り文化と同根。
5. 建物: 進捗を垂直軸+数値(30%)で表象（残り70%という暗黙の野心・楽観）。
   内面は手入れされた快適な空間。自己イメージの画角に他者が入っていない。
6. **圧縮本能**（訂正で判明した最強の発見）: 曖昧な指示に対し「個別に答える」ではなく
   「全部を貫く1つを探す」と解釈するのが認知のデフォルト。10語の交差点に「アンドロイド」
   という物語的容れ物を、6文の交差点に「この時代にいない」という自己規定を置けている —
   最大圧縮時に残るのがSF的自己像。抽象階段を昇る癖(#1)の刺激レベルでの再現。
   時間圧でも説明できない（圧縮解釈のほうが個別列挙より高コスト）ため頑健。
   ※「確認せず推測で進める傾向(F15人間版)」は本人指摘の交絡（実験者が速度優先・
   推敲禁止を課し確認質問を封じていた）により**格下げ・保留**。

## 追補: 昇りは反射、降りは労働（2026-07-07・本人の自己報告から）

「曖昧な指示を具体化するとき苦戦する」という自己報告。全データと整合する非対称性:
抽象方向（事例→原則・定義・不変量）は反射的、具体方向（曖昧な依頼→ケース・手順・
受け入れ条件）は高負荷。傍証として、本人の自作ハーネスの中核はすべて**具体化の補装具**
（要件skillの Case Shaping・確認セット、業務構造化の「机上でなく実況メモ」、テンプレ群）。

**AI相補性仮説**: 体系化スタイルが「AIを使い始めてから」出現した理由の説明候補。
LLMは強力な具体化(降り)エンジンであり、本人の昇り反射と正確に相補的。欠けた半身が
接続されたことで認知スタイルが完成し、オーケストレーター型に自然に収まった。

処方箋: (1) 頭の中でなく現実に当てて具体化する（実況メモ原則の自分適用）
(2) 3例ルール — 具体例を3つ挙げられなければ理解は未完了 (3) 降りはAIに分業し、
本人は展開結果の判定に回る。

# 検証可能な予測

1. 新しいチェックリストや手法を渡されたら「儀式化しないか・発火が保証されるか」を必ず問う。
2. 曖昧な標語（構造化・アジャイル等）に出会うと操作的定義を要求する。
3. 興味が点火した日は関連相談を数珠つなぎに5件以上連鎖させる（バースト学習）。
4. 良いアイデアの多くは机の前ではなく散歩・風呂・深夜に出る（→机で詰まったら離れるのが正解）。
5. 報告や文書では読み手への配慮（分かりやすさ・失礼のなさ）に工数を掛けすぎる傾向が出る。
6. 好きな作品の感想を書く場があれば継続する（感情の消化経路と一致するため）。
