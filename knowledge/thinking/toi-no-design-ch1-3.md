# 『問いのデザイン』第1〜3章の骨格

安斎勇樹・塩瀬隆之『問いのデザイン 創造的対話のファシリテーション』(学芸出版社, 2020)。
本書の全体構成は「課題のデザイン」(第2〜3章: 何を解くべきかを定める)と
「プロセスのデザイン」(第4章以降: どう解くかのワークショップ/ファシリテーション)の二層。
第1〜3章は前者 = **解くべき問題を正しく定める技術**。

## 本書の問題設定(前提)

組織・地域・学校が変われない本質は 2 つの病い:
- **認識の固定化** — 暗黙の前提・固定観念が深い理解と発想を阻む
- **関係性の固定化** — 役割・上下・対立が固定し、対話が起きない

問いはこの 2 つを「揺さぶって編み直す」ための媒体。良い問い→創造的対話→認識と関係性の更新、という機構。

## 第1章 問いのデザインとは何か

**問いの基本性質 7 つ**(要点: 問いは情報取得の道具ではなく、思考・感情・関係を駆動する装置):
1. 問いの設定によって、導かれる答えは変わりうる
2. 問いは、思考と感情を刺激する
3. 問いは、集団のコミュニケーションを誘発する
4. 対話を通して問いに向き合う過程で、個人の認識は内省される
5. 対話を通して問いに向き合う過程で、集団の関係性は再構築される
6. 問いは、創造的対話のトリガーになる
7. 問いは、創造的対話を通して、新たな問いを生み出す

**コミュニケーションの 4 タイプ**: 雑談(自由な関係構築) / 討論(正しさの決着) / 議論(合意形成・意思決定) / **対話**(前提を保留し、共通の意味づけを探る)。創造的対話 = 対話の過程で新しい意味やアイデアが創発するもの。討論は勝敗、対話は理解、が目的という違いが軸。

**基本サイクル**: 問いのデザイン → 創造的対話 → 認識・関係性の変化 → 新たな問い、の循環。
**デザイン手順**: まず課題をデザインし(第2〜3章)、次にプロセスをデザインする(第4章〜)。

## 第2章 問題を捉え直す考え方

**問題と課題の区別**(本書の中核概念):
- **問題** = 目標があり動機づけられているが、到達方法がわからない状況。当事者の解釈に依存し、人によって見え方が違う主観的なもの
- **課題** = 関係者の間で「解決すべきだ」と**合意された**問題

つまり課題設定とは客観的な発見ではなく、複数の主観をすり合わせて合意を作る行為。
洞察問題(9点問題のような、暗黙の制約を外さないと解けない問題)の例で、固定観念こそが解決を阻むことを示す。

**課題設定の罠 5 つ**:
1. **自分本位** — 当事者・相手の視点を欠いた設定
2. **自己目的化** — 手段(ワークショップ開催等)が目的にすり替わる
3. **ネガティブ・他責** — 「〜できていない」「〜のせい」は当事者意識と創造性を殺す
4. **優等生** — 正論だが切実さがなく、誰の心も動かさない
5. **壮大** — 大きすぎて手がつかず、自分ごとにならない

**問題を捉える思考法 5 つ**(認識の固定化を破るレンズ群):
1. **素朴思考** — 素朴な疑問・違和感をそのまま口に出す
2. **天邪鬼思考** — あえて逆張りし、通説・前提を疑う
3. **道具思考** — 理論・フレームワークを分析のレンズとして意図的に使い分ける
4. **構造化思考** — 要素と関係を図解し、全体構造の中に問題を位置づける
5. **哲学的思考** — 使っている概念そのものの定義を問い直す(「そもそも◯◯とは何か」)

## 第3章 課題を定義する手順(5 STEP)

1. **要件の確認** — 依頼主・関係者の要望、既存目標の有無、制約を確認する。曖昧なまま進めない
2. **目標の精緻化** — 3 軸で整形: 期間(短期/中期/長期) / 優先順位 / 目標の性質(**成果目標**=アウトプット・**プロセス目標**=過程のあり方・**ビジョン**=その先の展望)
3. **阻害要因の検討** — 目標と現状の間のハードルを特定する。ハードルこそが解くべき課題の候補であり、検討の過程で目標自体が修正されることもある。典型類型(記憶ベース・要原典確認): 対話の機会がない / 固定観念が強固 / 意見が割れて合意できない / 自分ごとになっていない / 知識・創造性の不足
4. **目標の再設定(リフレーミング)** — 技法 10: 利他的に考える / 大義を問い直す / 前向きに捉える / 規範外にはみだす / 小さく分割する / 動詞に言い換える / 言葉を定義する / 主体を変える / 時間尺度を変える / 第三の道を探る
5. **課題の定義** — 再設定した目標を、関係者が合意できる「解くべき課題」として言語化する。第2章の罠 5 つに照らしてチェックする

## 既存 Knowledge との接続

- 「問題=主観的解釈、課題=合意」の構図は、[不確実性の操作的分解](/thinking/uncertainty-reduction-as-operations.md)の「不確実性の源は未来と他人」の**他人側**を扱う技術と読める。課題設定 = 他人との認識のすり合わせを構造化する操作
