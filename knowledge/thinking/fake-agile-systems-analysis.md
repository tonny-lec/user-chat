# 相談内容

企業では「なんちゃってアジャイル」と呼ばれる、アジャイルでもウォーターフォールでもない開発体制ができあがってしまう。この構造をシステム思考で可視化できるか。

# 検討・調査

[システム思考の基礎](/thinking/systems-thinking.md) の概念（フィードバックループ・システム原型・レバレッジポイント）を適用。可視化した Artifact: https://claude.ai/code/artifact/802b8e0d-dd45-402c-ac6b-6aebf61e617a

## 見つけた構造（3つ）

1. **R: 管理強化の悪循環**（ウォーターフォール残存の理由）
   経営の不安 →＋ 管理・報告・承認 →＋ 固定スコープ・固定納期 →＋ 変更コスト →− 現場の学習 →＋ 成果と信頼 →− 経営の不安。
   不安への対処（管理強化）そのものが次の不安を製造する自己強化ループ。
2. **B: 「やってる感」ループ**（変革圧力を吸収する安全弁）
   変革への圧力 →＋ 儀式の導入（朝会・スプリント・レトロ）→＋ やってる感 →− 変革への圧力。
   圧力は最も安い対処＝儀式導入に流れ、「もうアジャイルです」の空気で鎮火する。
3. **欠けている配線**: 儀式 → 学習 の線は、権限委譲と変動スコープ契約がない限り存在しない。だから儀式を丁寧にやっても R ループは減速しない。

## システム原型としての同定

これは Senge の **Shifting the Burden（応急処置の失敗）** の教科書的実例。
- 症状: 開発が遅い・先が読めない
- 対症療法（速い・楽）: 儀式の導入。誰の権限も脅かさない
- 根本対策（遅い・痛い）: 権限委譲・変動スコープの契約・顧客と直接対話
- 副作用 R: 「もう導入済み」という既成事実が根本対策の議題を消す

# 結論

**なんちゃってアジャイルは失敗ではなく、システムの安定解（アトラクタ）**。経営の不安は管理強化で、変革圧力は儀式導入で、全員の不安が最も安く同時に解消される点で静止する。誰も怠けていない——構造が行動を生む。だから犯人探しは効かない。

## 介入点（レバレッジの弱い順）

1. パラメータ調整（スプリント長・会議形式）— ほぼ無効。「なんちゃって改善」の定番
2. **情報の流れ**: スプリントレビューに意思決定者と実顧客を入れ、動くソフトを直接見せる — 現実的な最初の一手。報告要求（R ループの燃料）を減らす
3. **ルール**: 固定スコープ×固定納期×固定予算の最低ひとつを変動化 — 欠けた配線がここで初めてつながる
4. 目的: 評価指標を「計画遵守率」から「学習の速さ・リードタイム」へ
5. パラダイム: 不確実性を「排除すべきリスク」から「学習の原料」へ — 最強で最難。R ループが逆回転を始める
