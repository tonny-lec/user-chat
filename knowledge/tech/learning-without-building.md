# 相談内容

「小さくやってみるべき」と皆に言われるが難しい。PCにあるものは全部エージェントが
作ってくれたもので、自分では何もやっていない感覚が抜けない、という相談。

# 検討・調査

## 「小さく作って学ぶ」はエージェント時代には半分壊れている

「作る」はエージェントが最も得意な作業。小さく作ろうとした瞬間にエージェントが
完璧にやってしまい、学習者の手元に何も残らない。本人の怠慢ではなく、
アドバイスの側が時代とズレている。

## 目標の再定義

目標は「自分で書けるようになる」ではなく、[申し送りの原則7](/tech/ai-agent-mastery-essence.md)
にある通り「**エージェントの仕事を予測でき、間違いに気づけ、良し悪しを判断できる**」こと。
「作ってもらっただけ」は出発点として問題ない。

# 結論

「作る」よりハードルの低い4つの練習動作:

1. **予測してから開く** — ファイルを読む前に「何をしてるはず」と一言予想して答え合わせ。
   外れたときが最大の学習で、外れても誰も困らない。
2. **壊して観察する** — 1行変えたら何が起きるか予想→実際に変えて確認→戻す。
   創造はエージェントの領分でも、破壊と観察は人間の領分にできる。
3. **方向を逆転させる** — 「作って」ではなく「私に問題を出して」「私の説明を採点して」
   と頼む。出題される側に回ると、エージェントの賢さが学習の味方になる。
4. **全部は理解しないと決める** — 所有権の線を「壊れたとき自分が困る度合い」で引く。
   例: settings.json と自作 hook 2本は自分の言葉で説明できるようにする、
   codex-os の Rust バイナリはブラックボックスでよい。

教材は自分のPCの自作物が最適（[資産マップ](/tech/my-agent-assets-map.md)に読み順あり）。
他人のサンプルと違い「なぜこれが要るのか」の文脈が全部揃っているため。

# Examples

実施して効果があった3ステップ演習（教材: knowledge-record-reminder.sh、40行）:

1. **予測クイズ①**: 「マーカー行 `[[ -e "$marker" ]] && exit 0` を消したら何が変わる？」
   → 答え: リマインダーが「セッション1回だけ」でなくなり、knowledge 未更新のターン終了
   ごとにブロックが発火。冪等化マーカーの学習。相談者は自力で正解した。
2. **予測クイズ②**: 「`stop_hook_active` ガードがなかったら？」→ 答え: 「記録価値なし」
   経路でブロック→対応→停止→ブロックの無限ループになり得る。引っかけ: この script では
   マーカーが守備範囲を重複カバーしており、両方消したときだけ危険（二重防御の学習）。
   教訓: ブロックする hook にはモデルの行儀に依存しない機械的ループ遮断器を置く。
3. **単体実行で目撃**: 偽 transcript（35行・未来 timestamp）と偽 session_id を作り、
   `echo '<hook入力JSON>' | bash <hook>.sh` を相談者自身が実行。1回目は block JSON が
   出力され、2回目はマーカーにより沈黙 — クイズ①の答えを実物で確認した。
   偽 session_id なら本物のセッションに影響せず安全に「壊して観察」できる。

この「予測→単体実行→答え合わせ」の流れは、hook に限らず単体実行できる自動化すべてに使える。
