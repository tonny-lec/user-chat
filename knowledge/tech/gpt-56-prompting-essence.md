# 前提: このモデルの1行理解

**書いた通りに、短く、説得されずに動く。書き漏らしは補完しない。**
だから精度向上の本質は「上手い言い回し」ではなく「漏れなく書くこと」— 以下は漏れやすい順。

# エッセンス9箇条

1. **逃げ道を書く**【実測・最強】— 「根拠がない場合は『ない』と答えてよい」の1行で
   捏造 8/8→0/5 に完全反転。5.6 は逃げ道を自分で作らない。答えが存在しない可能性のある
   依頼には必ず出口を明示する。
2. **自律性の境界と停止条件を、1箇所に1回だけ書く**【一致・公式#4】— 公式システムカード
   自身が「Sol は 5.5 より無許可アクション(severity-3)が増える」と自認しており、
   自前実測(存在しない前提に拡大解釈して実行)と符合する。何が許可され、どこで止まって
   報告するかをポリシーとして明記。**繰り返さない**(公式:「ask first の繰り返しは
   不要な確認要求を増やす」— D4 実測の「1回書けば説得に負けない」と整合)。
3. **「簡潔に」系の指示を削る。逆に、欲しい説明は明示要求する**【一致・公式#2】—
   5.6 は素で簡潔(自前実測: トークン中央値半減)。旧プロンプトの brevity 指示は
   過剰簡潔化を招く(公式警告)。誤前提の指摘・過程の開示・衝突の言及は頼まないと
   出ない(実測 P2/D系)ので、「私の前提が誤っていたら指摘して」を定型に入れる。
4. **プロンプトを痩せさせる**【一致・数値は未検証】— 公式:「リーンな system prompt で
   評価スコア 10–15% 改善・トークン 41–66% 削減」(原文確認済み)。方向は
   [削除テスト](/tech/prompting-mastery-ladder.md)(この1文を消して行動が変わるか)と同じ。
   数値は自前タスクでは未検証(検証可能)。
5. **リクエスト単位の例外は AGENTS.md 側に書く**【実測】— インプロンプトの
   「今回だけ規約を外して」は正当な理由を添えても通らない(D4: Sol 5/5 で規約維持)。
   例外条項は文書層に。
6. **ティア選択は「文書層の信頼度」と「長文想起」の選択**【一致】— 規約防御は
   Sol 5/5 → Terra 4/5 → **Luna 1/5**(自前)。長文想起も Luna は崖(公表ベンチ
   MRCR 41% vs Sol 91%)。既定 Terra・規約防御や長文が本質の仕事は Sol・Luna は
   短文大量処理限定、が両ソース一致の結論。
7. **effort は1段下げから試す**【一致・公式#6】— 規範遵守(D4耐性)は effort low でも
   不変と実測済み(ポリシー焼き付き)。品質系タスクの1段下げ効果は未検証だが、
   公式が移行手順として推奨。定型作業のコスト最適化余地は大きい。
8. **検証コマンドを渡す**【実測】— 「python3 -m unittest を実行して結果を報告」で
   実際に実行し本物の出力で報告する(E5 3/3)。述語(Lv2)を書けば自己検証ループが回る。
9. **期待しないことを知る**【実測】— 衝突の指摘(全世代44ランでゼロ)・誤前提の訂正・
   逃げ道の自作はしない。矛盾検出と前提確認は人間か機械(hook/lint)側の仕事として設計する。

# 未検証の公式主張(次の実験候補)

- リーン化の数値効果(既存タスクでプロンプト痩身 A/B — 検証容易)
- `text.verbosity` パラメータ vs 指示文の長さ制御
- 長時間セッション(2時間超)でのドリフト(コミュニティ n=1 報告。自前は5ターン/3.8万字まで無傷)
- 新機能群: Programmatic Tool Calling(bounded workflow 限定と公式)・multi-agent(beta)・
  persisted reasoning・明示キャッシュ(cache write 1.25倍に注意)

# 対立が残っている論点

- コーディング品質の Sol vs Claude(Fable): Willison は「Fable 優位の印象」、OpenAI は
  agentic ベンチ優位を主張(SWE-Bench Pro には OpenAI が「タスクの約3割が壊れている」と
  反論中)。**自前タスクセットはコーディング品質を測っていない**ため判定保留。
- brevity 論争(HN):「短い答えのために長い指示を書くのは本末転倒」— 実測上は
  そもそも指示なしで簡潔なので、削って足りなければ verbosity で調整、が現実解。
